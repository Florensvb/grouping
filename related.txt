Coin Shuffle - https://allquantor.at/blockchainbib/pdf/ruffing2014coinshuffle.pdf
  > On chain mixing
  > Generate new addresses every round
  > Shuffle addresses
  > Make sure amount of input and output coins are equal

Decentralizing Privacy: Using Blockchain to Protect Personal Data - http://homepage.divms.uiowa.edu/~ghosh/blockchain.pdf
  > Fine-grained access control for user data in for example mobile applications
  > Stores files in an off-chain DHT

k-ANONYMITY: A MODEL FOR PROTECTING PRIVACY - https://www.win.tue.nl/~jhartog/CourseVerif/Papers/10.1.1.90.4099.pdf
  > If there are k duplicate rows in a table then you have k-anonymity
  > Queries always yield at least k individuals
  > Attacks: Order of rows, linking rows to external data, temporal inference attack -> over time tracking of cruds
    -> Could use same attack vectors thesis evaluation
  > Also known as plausible deniability
